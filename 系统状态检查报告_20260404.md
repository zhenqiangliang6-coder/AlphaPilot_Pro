**修复措施**:
已完整实现所有缺失方法，共计 **350+ 行代码**，包括：

#### A. 信号处理流程
```python
def process_signal(self, code, action, price, volume_ratio):
    """三级过滤机制"""
    1. 严格拒绝 SELL 信号
    2. 检查股票类型是否为 delayed
    3. 量比阈值过滤（min_volume_ratio）
```

#### B. 观察名单管理
```python
def check_and_execute(self):
    """每日轮询观察名单"""
    1. 读取 watchlist.json
    2. 遍历每只股票
    3. 判断当前日期 >= target_date
    4. 到达则执行买入
    5. 成功后从名单移除
```

#### C. 买入执行（实时价格优先）
```python
def _execute_buy(self, code, item):
    """遵循 V8.5 资金管理规范"""
    1. 获取实时行情（lastPrice）

    3. 单次买入不超过 50000 元
    4. 最小订单金额 15000 元
    5. 下单价格 = 实时价 * 1.01（固定溢价）
    6. 涨停价保护
```

#### D. 目标日期计算（跳过周末）
```python
def _calculate_target_date(self, signal_date, delay_days):
    """自然日计算，自动跳过周末"""
    while remaining > 0:
```

## 九、潜在风险与建议

### ⚠️ 9.1 观察名单膨胀风险

**风险描述**: 
如果大量股票加入观察名单但未及时清理，可能导致：
- JSON 文件过大，读取缓慢
- 内存占用增加
- 无效轮询浪费资源

**建议措施**:
```python
# 在 check_and_execute() 中添加过期清理
MAX_WAIT_DAYS = 10  # 最大等待天数

for code, item in watchlist.items():
    signal_date = datetime.datetime.strptime(item['signal_date'], '%Y-%m-%d').date()
    wait_days = (today - signal_date).days
    
    if wait_days > MAX_WAIT_DAYS:
        logger.log("[清理] " + str(code) + " 等待超时，从名单移除")
        codes_to_remove.append(code)
```

### ⚠️ 9.2 节假日处理缺失

**风险描述**:

**优先级**: 中（可在节后统一调整）

### ⚠️ 9.3 并发写入冲突

**风险描述**: 
虽然使用了原子写入，但如果多个进程同时修改 `delayed_watchlist.json`，仍可能丢失数据

**建议措施**:
```python
import fcntl  # Linux/Mac
# 或
import msvcrt  # Windows

def _save_watchlist(self, data=None):
    temp_file = self.watchlist_file + ".tmp"
    
    with open(temp_file, 'w', encoding='utf-8') as f:
        # Windows 文件锁
        msvcrt.locking(f.fileno(), msvcrt.LK_LOCK, 1)
        json.dump(data, f, ensure_ascii=False, indent=2)
        msvcrt.locking(f.fileno(), msvcrt.LK_UNLCK, 1)
    
    os.rename(temp_file, self.watchlist_file)
```

**实际情况**: 当前单进程架构下风险较低，可作为未来扩展预留
