### ⚠️ 2. 资产查询空值风险（已修复）

**问题描述**: 
`query_asset()` 可能返回 `None`，虽然使用了 `getattr(asset, 'cash', 0)`，但如果 `asset` 本身是 `None`，在某些极端情况下可能导致不可预期的行为。

**原代码**:
```python
asset = self.engine.query_asset()
if not asset:
    if logger:
        logger.log("[错误] " + str(code) + " 资产查询失败")
    return False

available_cash = getattr(asset, 'cash', 0) * 0.98
```

**问题分析**:
- 虽然有 `if not asset:` 判断，但逻辑不够明确
- 缺少对 `None` 值的显式处理说明

**修复方案**:
```python
# === 重点修复：资产查询为空的情况 ===
asset = self.engine.query_asset()
if not asset:
    if logger:
        logger.log("[错误] " + str(code) + " 资产查询返回None，跳过买入")
    return False

available_cash = getattr(asset, 'cash', 0) * 0.98  # 保留2%缓冲
```

**改进点**:
- ✅ 明确注释标注"重点修复"
- ✅ 日志信息更具体（"返回None"而非笼统的"失败"）
- ✅ 确保在 `asset` 为 `None` 时立即返回，避免后续计算

### ⚠️ 3. 行情获取异常处理增强（已修复）

**问题描述**: 
`get_tick_data()` 在网络波动或接口异常时可能抛出异常，原代码仅在函数级别的 `try-except` 中捕获，不够精细。

**原代码**:
```python
try:
    ticks = self.engine.get_tick_data([code])
    
    if ticks and code in ticks:
        current_price = ticks[code].get('lastPrice', 0)
        limit_up = ticks[code].get('highLimit', 0)
    else:
        # 降级逻辑
        ...
except Exception as e:
    # 整体异常处理
    ...
```

**风险分析**:
- 如果 `get_tick_data()` 内部抛出异常，会直接跳到外层 `except`
- 无法区分是"行情获取失败"还是"买入执行失败"
- 日志信息不够精确

**修复方案**:
```python
# === 获取实时行情（增强异常处理）===
ticks = None
try:
    ticks = self.engine.get_tick_data([code])
except Exception as e:
    if logger:
        logger.log("[行情] 获取Tick数据异常：" + str(e) + "，使用触发价")
    ticks = None

current_price = 0
limit_up = 0

if ticks and code in ticks:
    current_price = ticks[code].get('lastPrice', 0)
    limit_up = ticks[code].get('highLimit', 0)
    
    if current_price <= 0:
        if logger:
            logger.log("[警告] " + str(code) + " 实时价格为0，使用触发价")
        current_price = item.get('trigger_price', 0)
else:
    if logger:
        logger.log("[警告] " + str(code) + " 无法获取实时行情，使用触发价")
    current_price = item.get('trigger_price', 0)
```

**改进点**:
- ✅ 独立的 `try-except` 块专门处理行情获取异常
- ✅ 明确的日志分类：`[行情]` vs `[警告]` vs `[错误]`
- ✅ 即使 `get_tick_data()` 抛异常，也能优雅降级到触发价
- ✅ 初始化 `current_price = 0` 和 `limit_up = 0`，避免未定义变量
