## 资金管理流程

### V8.5 标准买入流程：

```
1. 查询可用现金：100000 元
2. 保留 2% 缓冲：100000 × 0.98 = 98000 元
3. 使用 80% 仓位：98000 × 0.8 = 78400 元
4. 检查上限：min(78400, 50000) = 50000 元
5. 计算股数：50000 / 42.15 = 1186 股 → 1100 股（向下取整）
6. 特殊处理：如果计算结果<100 但钱够，强制买 100 股
7. 最终校验：1100 × 42.15 = 46365 元 < 98000 ✓
```

### 特殊情况处理：

#### 场景 1：现金极度紧张
```
可用现金：12000 元
保留缓冲：12000 × 0.98 = 11760 元
目标金额：11760 × 0.8 = 9408 元 (< 15000 最低)
→ 使用全部现金：9408 元
→ 计算股数：9408 / 42.15 = 223 股 → 200 股
→ 订单金额：200 × 42.15 = 8430 元 (< 15000)
→ 跳过（避免无效委托）
```

#### 场景 2：现金足够但计算结果不足 100 股
```
股价：480 元/股
目标金额：50000 元
计算股数：50000 / 480 = 104 股 → 100 股 ✓
（虽然少，但达到最小交易单位，允许）
```

## 风险控制机制

### 1. **现金缓冲** (2%)
```python
available_cash = asset.cash * 0.98
```
- 防止计算误差导致透支
- 应对突发费用（如印花税调整）

### 2. **单笔上限** (5 万元)
```python
if target_cash > 50000.0:
    target_cash = 50000.0
```
- 避免过度集中单只股票
- 分散投资风险

### 3. **最小订单金额** (15000 元)
```python
if order_value < 15000:
    continue  # 跳过
```
- 过滤低效交易
- 减少手续费占比

### 4. **重复保护** (9 分钟)
```python
REPEAT_PROTECT_SECONDS = 540
```
- 防止重复下单
- 避免误操作
