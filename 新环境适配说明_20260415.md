## ⚠️ 重要注意事项

### 1. Python解释器位置

新环境中Python可能位于以下任一位置（按优先级）：
1. `bin.x64\python.exe` ← **首选**（新标准）
2. `mpython\python.exe`
3. `python\python.exe`

启动脚本会自动检测，无需手动干预。

### 2. QMT客户端登录

**必须保持QMT客户端登录在线**，否则无法连接交易接口。

### 3. 代码更新流程

每次修改代码后：
1. 停止正在运行的程序（Ctrl+C）
2. 重新启动（双击 `启动AlphaPilot.bat`）
3. 脚本会自动清理缓存

### 4. 跨版本兼容性

本项目的路径设计采用**相对路径 + 动态获取**机制，理论上可以在任何QMT版本中运行，只需修改 `config/settings.py` 中的 `QMT_PATH` 即可。

## 🛠️ 故障排查

### 问题 1：找不到 Python 解释器

**症状**：启动脚本显示 `❌ 错误：未找到 QMT Python 解释器！`

**解决**：
1. 检查QMT是否正确安装
2. 手动查找Python位置：
   ```batch
   dir "D:\迅投极速交易终端 睿智融科版" /s /b | findstr python.exe
   ```
3. 编辑启动脚本，添加正确的路径

### 问题 2：xtquant 模块不可用

**症状**：显示 `ModuleNotFoundError: No module named 'xtquant'`

**解决**：
1. 确认使用的是QMT自带的Python，不是系统Python
2. 检查QMT客户端是否已登录
3. 尝试其他Python路径（mpython 或 python 文件夹）

### 问题 3：路径相关错误

**症状**：程序启动后立即退出或报错

**解决**：
1. 运行路径验证命令（见上方"环境验证"部分）
2. 检查 `config/settings.py` 中的路径配置
3. 确认工作目录正确（应为 `D:\AlphaPilot_Pro`）
