## 🚀 一分钟上手

### 1. 配置慢涨股
编辑 `data/stock_personalities.json`：
```json
{
    "603538": {
        "name": "美诺华",
        "type": "delayed",
        "delay_days": 1,
        "min_volume_ratio": 18.0,
        "target_day_min_vr": 0.2
    }
}
```

### 2. 发送信号
在信号目录放置文件（每行一个 JSON）：
```json
{"code": "SH603538", "action": "BUY", "price": 40.0, "volume_ratio": 20.0}
```

### 3. 自动执行
- ✅ 信号量比 >= 18.0 → 加入观察名单
- ✅ 等待 1 天（跳过周末）
- ✅ 到达目标日 → 自动买入

---

## 📊 核心参数速查

| 参数 | 含义 | 推荐值 | 位置 |
|------|------|--------|------|
| `type` | 股票类型 | "delayed" / "immediate" | stock_personalities.json |
| `delay_days` | 延时天数 | 1-3 天 | stock_personalities.json |
| `min_volume_ratio` | 等待期量比门槛 | 15.0-20.0 | stock_personalities.json |
| `target_day_min_vr` | 目标日量比门槛 | 0.2-0.5 | stock_personalities.json |
| `FIXED_ORDER_AMOUNT` | 单笔买入上限 | 50000 元 | settings.py |
| `MIN_ORDER_VALUE` | 最小订单金额 | 15000 元 | settings.py |

## ⚠️ 注意事项

### ✅ 必须遵守
1. **只配置 BUY 信号** - SELL 信号会被严格拦截
2. **量比要达标** - 等待期门槛高（如 18.0），目标日门槛低（如 0.2）
3. **耐心等待** - 按自然日计算，周末自动跳过
4. **不要手动编辑 watchlist** - 由系统自动维护

### ❌ 禁止操作
1. ~~手动删除 delayed_watchlist.json~~ → 会导致断点续传失效
2. ~~在等待期内频繁修改配置~~ → 可能导致逻辑混乱
3. ~~设置过低的 min_volume_ratio~~ → 会失去过滤假突破的意义

## 🐛 故障排查

### Q1: 信号没有被加入观察名单？
**检查清单**:
- [ ] 动作是否为 "BUY"？
- [ ] `type` 是否设置为 "delayed"？
- [ ] 量比是否 >= `min_volume_ratio`？
- [ ] 日志中是否有 "[延时过滤]" 提示？

**解决**: 查看日志，找到具体被过滤的原因

---

### Q2: 到达目标日期没有买入？
**检查清单**:
- [ ] 当前日期是否真的 >= target_date？（注意周末顺延）
- [ ] 账户现金是否充足？（至少 15000 元）
- [ ] 是否在交易时间？（09:30-11:30, 13:00-15:00）
- [ ] 日志中是否有 "[延时策略] ★ 到达目标日期"？

**解决**: 检查日志中的错误信息，常见原因：
- 现金不足 → 充值或等待卖出回款
- 非交易时间 → 等待开盘
- 获取行情失败 → 检查网络连接

---

### Q3: 如何确认策略正在运行？
**验证方法**:
```bash
# 1. 查看日志最新输出
tail -f logs/run_2026-04-04.log | grep "延时策略"

# 2. 检查观察名单文件
cat data/delayed_watchlist.json

# 3. 发送测试信号
echo '{"code":"SH603538","action":"BUY","price":40.0,"volume_ratio":20.0}' > signals/test.txt
```

**预期日志**:
```
[延时策略] 603538 通过过滤：VR=20.0 >= 18.0
[延时策略] ✓ 603538 已加入观察名单
[延时策略]   信号日期：2026-04-04 -> 目标日期：2026-04-07
```

---

### Q4: 想提前买入怎么办？
**方法 1**: 等待期内再次出现强信号（VR 更高）会自动提前触发  
**方法 2**: 手动修改 `delay_days` 为更小的值（如从 3 改为 1）  
**方法 3**: 从观察名单中删除该股票，重新发送信号

## 📈 实战技巧

### 技巧 1: 分级量比设置
```json
{
    // 强势慢涨股：等待期高门槛，确保真突破
    "603538": {
        "min_volume_ratio": 18.0,
        "target_day_min_vr": 0.2
    },
    
    // 温和慢涨股：等待期中门槛，平衡确认与机会
    "300461": {
        "min_volume_ratio": 10.0,
        "target_day_min_vr": 0.2
    }
}
```

### 技巧 2: 动态调整延时天数
```python
# 根据历史表现调整
if 该股经常在第 2 天涨停:
    delay_days = 1  # 提前一天埋伏
elif 该股需要长时间洗盘:
    delay_days = 3  # 给予充分整理时间
```

### 技巧 3: 结合大盘环境
```
大盘上涨趋势 → 可降低 min_volume_ratio（如 15.0）
大盘震荡整理 → 保持标准门槛（如 18.0）
大盘下跌趋势 → 提高门槛或暂停延时策略（如 25.0）
```
