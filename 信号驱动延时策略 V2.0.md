# 信号驱动型延时策略 - V2.0

## 🎯 核心创新

### 传统延时策略（时间驱动）❌
```
Day 1: 收到信号 → 加入观察名单 → 等待 N 天
Day N: 时间到了 → 机械式买入（不管市场情况）

问题：
- 死板教条，不够灵活
- 可能错过中间的最佳买点
- 第 2 天涨停也买不到
```

### 信号驱动型延时策略（V2.0）✅
```
Day 1: 收到信号 → 加入观察名单 → 开始监听
Day 2: 再次出现强势信号 → 立即买入（提前上车）⭐
Day 3: 如果还没买，时间到了再买入

优势：
- ✅ 抓住最低点（第 2 天有信号说明主力行动）
- ✅ 双重确认（时间 + 信号双重验证）
- ✅ 灵活智能（不机械执行）
```

**关键参数**：

- `min_volume_ratio`: 量比阈值（必须达到才触发）

---

### 2. 核心流程

```python
def process_signal(code, action, price, volume_ratio):
    """处理新信号"""
    
    # 1. 检查是否已在观察名单中
    if code in watchlist:
        days_waited = today - signal_date
        
        # 2. 如果未到目标日期且 >0 天 → 提前触发
        if 0 < days_waited < delay_days:
            log.log("等待第" + str(days_waited) + "天再次出现强势信号！")
            
            # 3. 量比校验
            if volume_ratio >= min_vr:
                # 4. 立即买入
                _execute_early_buy(code, price, "SIGNAL_TRIGGERED")
                return None  # 已触发
    
    # 5. 否则正常加入观察名单
    return True
```

---

### 3. 信号监听机制

```python
def process_recent_signals():
    """监听今日新信号，用于提前触发"""
    
    # 1. 读取今日所有信号文件
    files = get_today_signal_files()
    
    # 2. 遍历信号
    for file in files:
        signals = parse_signals(file)
        
        for sig in signals:
            code = sig['code']
            
            # 3. 只处理已在观察名单中的股票
            if code not in watchlist:
                continue
            
            # 4. 检查是否已过期
            if today >= target_date:
                continue
            
            # 5. 量比校验
            if sig['volume_ratio'] >= min_vr:
                # 6. 提前触发买入
                _execute_early_buy(code, sig['price'], "NEW_SIGNAL")
```

### 4. 防重复触发机制

```python
# 观察名单数据结构增强

    'signal_date': '2026-04-01',

    'trigger_price': 40.59,
    'status': 'waiting',
    
    # 【新增】防止重复触发

    'early_trigger_date': '',       # 提前触发日期

}
```

**保护逻辑**：
```python
if data.get('early_triggered', False):
    continue  # 已经触发过，跳过
```

```python

    return False  # 拒绝进入名单

    continue  # 量比不够，不触发
```


## 📞 故障排查

### 问题 1：为什么不提前触发？

**检查清单**：
1. ✅ 股票是否在观察名单中
2. ✅ 是否未到目标日期
3. ✅ 量比是否达到阈值
4. ✅ 是否有新的 BUY 信号

**调试方法**：
```bash
# 查看详细日志
grep "监听今日新信号" logs/run_*.log
grep "等待第.*天再次出现" logs/run_*.log
```

---

### 问题 2：重复触发怎么办？

**原因**：标记未正确设置  
**解决**：检查 `_save_watchlist()` 是否成功

**防御机制**：
```python
# 每次触发后立即保存
if buy_result:
    del watchlist[code]
    self._save_watchlist()  # 确保写入磁盘
```
