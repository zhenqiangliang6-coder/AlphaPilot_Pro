# 动态止盈模块使用说明

## 📋 **模块概述**

动态止盈模块是一个**完全独立**的三级止盈策略，专门用于自动卖出获利的持仓股票。

**关键特性**：
- ✅ 完全独立于买入策略和其他卖出策略
- ✅ 不影响其他策略的正常执行
- ✅ 自动追踪所有持仓的最高涨幅
- ✅ 三级止盈机制，适应不同股票类型

---

## 🎯 **三级止盈策略**

### **第一级：快速止盈（所有股票）**
- **触发条件**：上涨 3% 后回落 1.3%
- **执行动作**：立即卖出
- **适用场景**：短线快速获利了结
- **示例日志**：
  ```
  [止盈-快速] 600726.SH 触发第一级止盈 (最高涨幅: 4.50%, 当前涨幅: 3.15%, 回落: 1.35%)
  [止盈执行] 600726.SH 卖出 2600 股 @ 5.21 (止盈-快速(3%回落1.3%))
  [止盈成功] 600726.SH 已卖出，原因: 止盈-快速(3%回落1.3%)
  ```

### **第二级：波段止盈（60/00 开头股票）**
- **触发条件**：上涨 9% 后持有 12 分钟
- **执行动作**：12 分钟后卖出
- **适用场景**：主板股票（60xxxx/00xxxx）的波段操作
- **示例日志**：
  ```
  [止盈-波段] 600821.SH 首次达到第二级阈值 (涨幅: 9.20%)，开始计时 12 分钟
  [止盈-波段] 600821.SH 触发第二级止盈 (涨幅: 9.50%, 持有时间: 12.3 分钟)
  [止盈执行] 600821.SH 卖出 4700 股 @ 8.67 (止盈-波段(9%持有12分钟))
  ```

### **第三级：强势股止盈（68/30 开头股票）**
- **触发条件**：上涨 18% 后持有 12 分钟
- **执行动作**：12 分钟后卖出
- **适用场景**：科创板（688xxx）/创业板（300xxx）的强势股
- **示例日志**：
  ```
  [止盈-强势] 300763.SZ 首次达到第三级阈值 (涨幅: 18.50%)，开始计时 12 分钟
  [止盈-强势] 300763.SZ 触发第三级止盈 (涨幅: 19.20%, 持有时间: 12.1 分钟)
  [止盈执行] 300763.SZ 卖出 1100 股 @ 106.50 (止盈-强势(18%持有12分钟))
  ```

## 🔧 **参数配置**

所有参数在 `risk/dynamic_take_profit.py` 的 `__init__` 方法中：

```python
# 第一级：所有股票
self.level1_gain_threshold = 0.03    # 上涨 3%（可调：0.02~0.05）
self.level1_gain_max = 0.65          # 涨幅上限 65%（超过此值不执行第一级，交由第二/三级处理）
self.level1_drop_threshold = 0.013   # 回落 1.3%（可调：0.01~0.02）

# 第二级：60/00 开头股票
self.level2_gain_threshold = 0.083    # 上涨 9%（可调：0.07~0.12）
self.level2_hold_minutes = 12        # 持有 12 分钟（可调：5~20）

# 第三级：68/30 开头股票
self.level3_gain_threshold = 0.18    # 上涨 18%（可调：0.15~0.25）
self.level3_hold_minutes = 12        # 持有 12 分钟（可调：5~20）
```

**调整建议**：
- **更激进**：降低涨幅门槛，缩短持有时间
- **更保守**：提高涨幅门槛，延长持有时间
- **涨幅上限**：如果希望大涨的股票继续持有，可以提高 `level1_gain_max`（如 0.80 = 80%）

## 📊 **工作流程**

```
持仓股票
   ↓
[每 15 秒检查一次]
   ↓
计算实时盈亏比例
   ↓
更新最高涨幅记录
   ↓
检查三级止盈条件
   ├─ 第一级：涨 3%~65% 回落 1.3% → 立即卖出（超过 65% 不执行）
   ├─ 第二级：60/00 涨 9% 持 12 分钟 → 卖出
   └─ 第三级：68/30 涨 18% 持 12 分钟 → 卖出
   ↓
卖出成功后从追踪列表移除
```

## ⚠️ **注意事项**

1. **优先级顺序**：
   - 第一级止盈优先级最高（快速止盈）
   - **涨幅限制**：第一级只在 3% ~ 65% 范围内执行，超过 65% 交由第二/三级处理
   - 如果触发第一级，不会继续检查第二、第三级
   - 各级止盈互斥，每只股票每个级别只触发一次

2. **涨幅上限的作用**：
   - 防止大涨股票（如涨幅 > 65%）被第一级止盈过早卖出
   - 让强势股有第二级（9%）或第三级（18%）的机会继续持有
   - 可根据市场情况调整 `level1_gain_max` 参数（默认 0.65 = 65%）

3. **卖出价格**：
   - 止盈卖出价格 = 当前价格 × 0.99
   - 略微让利提高成交率
   - 如需调整，修改 `_execute_sell` 方法中的 `sell_price` 计算

4. **重新买入**：
   - 止盈卖出后如果重新买入，会自动重新开始追踪
   - 无需手动重置追踪器

5. **与其他策略的关系**：
   - ✅ 止盈卖出不影响买入策略（买入策略继续执行）
   - ✅ 止盈卖出不影响延时策略（延时策略继续等待）
   - ✅ 止盈卖出不影响其他卖出策略（如信号 SELL）
