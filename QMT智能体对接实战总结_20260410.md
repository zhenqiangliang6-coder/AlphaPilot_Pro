##  项目背景

AlphaPilot Pro 是一个基于信号驱动的自动化量化交易策略系统，核心功能包括：

- **信号接收与处理**：从外部信号源（TXT/JSON 文件）读取交易信号
- **延时策略**：支持个股个性化配置，实现"信号触发后延迟 N 天买入"
- **动态止盈**：三级止盈架构（快速/波段/强势），独立于买卖策略运行
- **硬性止损**：基于跌幅和时间窗口的风险控制
- **资金管理**：单笔订单上限、现金缓冲、最小订单门槛

**对接目标：** 将系统无缝集成到 QMT 交易终端，实现全自动化的信号处理、行情获取、仓位管理和下单执行。

## ⚠️ 对接 QMT 的关键挑战

### 1. Python 3.6 环境限制

**问题：** QMT 内置 Python 3.6，不支持现代语法（f-string、walrus operator `:=`）。

**解决方案：**
- 全局使用 `.format()` 或 `%` 进行字符串格式化
- 严禁使用 Emoji 和非 ASCII 字符（除非声明 `# -*- coding: utf-8 -*-`）
- 模块级别严禁调用 get_logger()，必须在函数内部动态获取

### 2. 文件系统与路径管理

**问题：** QMT 启动时工作目录不确定，导致相对路径失效。

**解决方案：**
- 在 main.py 开头强制切换目录：
  ```python
  os.chdir(os.path.dirname(os.path.abspath(__file__)))
  ```
- 所有关键路径在 config/settings.py 中使用绝对路径配置

### 3. 缓存导致的代码不生效

**问题：** 修改 .py 文件后，QMT 仍运行旧逻辑。

**根本原因：** Python 的 `__pycache__` 机制 + QMT 文件系统同步延迟。

**解决方案：**
- 更新代码后，**必须手动删除所有 `__pycache__` 文件夹**
- **完全关闭并重启 QMT 客户端**（仅停止策略无效）

### 4. 行情接口"假死"问题

**问题：** 长时间运行后，`xtdata.get_full_tick` 返回空字典 `{}`，不报错但无数据。

**解决方案：**
- 在 get_tick_data 中增加 `xtdata.subscribe_whole_quote` 强制刷新连接
- 主循环每 50 分钟执行 `gc.collect()` 清理内存碎片
- 增加空数据警告日志，便于排查

### 5. 配置文件格式错误

**问题：** 用户提供的是 Python 字典格式，导致 `json.load()` 报错：
```
[错误] 读取配置文件失败: Expecting value: line 1 column 1
```

**解决方案：**
- 将 Python 字典转换为标准 JSON 格式（移除 `#` 注释、`self.stock_config =` 等语法）
- 确保 stock_personalities.json 符合 RFC 8259 规范

## 💡 专家经验总结

### 1. QMT 对接的"三大铁律"

1. **永远不要信任文件系统同步**：修改代码后，清除缓存 + 完全重启是铁律。
2. **行情接口必须主动保活**：`xtdata` 不会自动重连，必须定期调用订阅接口。
3. **配置与状态必须分离**：静态参数用 JSON（热更新），动态状态用独立文件（持久化）。

### 2. 智能体协作的价值

- **代码质量保障**：AI 智能体自动检测 Python 3.6 兼容性问题，避免手动遗漏。
- **逻辑闭环验证**：通过符号搜索和语义分析，发现双重写入、时间边界等隐蔽 bug。
- **快速迭代优化**：从发现问题到修复上线，平均耗时 < 10 分钟。

### 3. 实战中的"坑"与"解"

| 坑 | 解 |
|---|---|
| QMT 日志不显示 print 输出 | 必须用自定义 Logger 写入文件 |
| 策略停止后代码不生效 | 必须完全关闭 QMT 客户端 |
| JSON 配置文件报错 | 检查是否包含 Python 特有语法 |
| 行情数据偶尔为空 | 增加降级策略（使用历史价格） |
| 止盈触发时间不准确 | 严格校验 `_can_execute_now()` |
