## 📋 功能概述

自动化调度器是一个智能任务管理系统，能够：

✅ **自动执行**：每个工作日（周一到周五）早上 8:30 自动启动系统  
✅ **顺序执行**：按 1→2→3 的顺序依次运行三个启动脚本  
✅ **周末跳过**：周六、周日自动跳过，不执行任何操作  
✅ **后台运行**：调度器在后台持续运行，无需人工干预  
✅ **日志记录**：完整记录所有执行过程和状态  
✅ **异常恢复**：遇到错误自动重试，确保系统稳定性

## 📊 工作原理

### 执行流程

```
启动调度器
    ↓
验证三个 BAT 文件是否存在
    ↓
进入主循环（持续运行）
    ↓
┌─────────────────────────────┐
│ 每分钟检查一次：             │
│                             │
│ 1. 今天是工作日吗？         │
│    - 是 → 继续              │
│    - 否 → 跳到明天          │
│                             │
│ 2. 当前时间 >= 8:30 吗？    │
│    - 是 → 执行启动序列      │
│    - 否 → 继续等待          │
└─────────────────────────────┘
    ↓
执行启动序列（仅在工作日 8:30）
    ↓
步骤 1: 运行 1_启动QMT.bat
    ↓ 
    ⏱️ 等待 90 秒（确保 QMT 完全就绪）
    ↓
步骤 2: 运行 2_启动监听.bat
    ↓
    ⏱️ 等待 90 秒（确保监听模块完全就绪）
    ↓
步骤 3: 运行 3_AlphaPilot.bat
    ↓
等待到明天的 8:30
    ↓
回到主循环
```

**总启动时间**：约 3 分钟（90 秒 × 2 次间隔）

---

### 为什么需要 90 秒间隔？

#### **阶段 1：QMT 启动（90 秒）**
- ✅ 加载交易接口模块
- ✅ 建立与券商服务器的连接
- ✅ 初始化行情数据订阅
- ✅ 验证账户信息和权限
- ✅ 确保交易通道完全就绪

#### **阶段 2：信号监听启动（90 秒）**
- ✅ 建立邮件服务器连接
- ✅ 开始监听信号文件目录
- ✅ 初始化信号解析器
- ✅ 测试信号处理流程
- ✅ 确保信号接收稳定

#### **阶段 3：主策略启动（无需等待）**
- ✅ 自动检测 QMT 和监听模块状态
- ✅ 初始化所有策略模块（延时策略、止盈止损等）
- ✅ 建立稳定的行情订阅
- ✅ 开始执行交易逻辑

## ⚙️ 配置说明

### 修改执行时间

如果需要调整执行时间（例如改为 9:00），编辑 [`auto_scheduler.py`](file://d:\AlphaPilot_Pro\auto_scheduler.py) 文件：

```python
# 找到第 18-19 行
SCHEDULE_HOUR = 8      # 小时（0-23）
SCHEDULE_MINUTE = 30   # 分钟（0-59）

# 修改为 9:00
SCHEDULE_HOUR = 9
SCHEDULE_MINUTE = 0
```

### 修改检查间隔

默认每 60 秒检查一次时间，如需调整：

```python
# 找到第 33 行
CHECK_INTERVAL = 60  # 秒

# 修改为 30 秒（更频繁）
CHECK_INTERVAL = 30
```

### 修改脚本路径

如果三个 BAT 文件的路径发生变化，编辑第 24-28 行：

```python
BAT_FILES = [
    os.path.join(PROJECT_DIR, "1_启动QMT.bat"),
    os.path.join(PROJECT_DIR, "2_启动监听.bat"),
    os.path.join(PROJECT_DIR, "3_AlphaPilot.bat"),
]
```

## ⚠️ 注意事项

### 1. 保持调度器运行

- ✅ **推荐**：将调度器窗口最小化，让它持续在后台运行
- ❌ **禁止**：关闭窗口或按下 Ctrl+C，否则调度器会停止

### 2. 开机自启动（可选）

如果需要电脑重启后自动启动调度器，可以：

**方法 1：添加到启动文件夹**
1. 按 `Win + R`，输入 `shell:startup`
2. 将 `启动自动调度.bat` 的快捷方式复制到该文件夹

**方法 2：使用任务计划程序**
1. 打开"任务计划程序"
2. 创建基本任务，触发器设为"登录时"
3. 操作设为"启动程序"，选择 `启动自动调度.bat`

### 3. 周末和节假日

- 调度器会自动跳过周六、周日
- 法定节假日需要手动停止调度器（按 Ctrl+C）

### 4. 异常情况处理

如果遇到以下情况：

| 异常 | 原因 | 解决方法 |
|------|------|----------|
| 脚本未找到 | BAT 文件被移动或删除 | 检查文件路径，恢复文件 |
| QMT 未启动 | QMT 进程启动失败 | 检查 QMT 安装路径 |
| 端口占用 | 多个实例同时运行 | 关闭重复的调度器窗口 |

## 🛠️ 故障排除

### 问题 1：调度器无法启动

**症状**：双击 `启动自动调度.bat` 后立即关闭

**解决**：
1. 检查 QMT Python 路径是否正确
2. 编辑 `启动自动调度.bat`，确认第 16 行的路径：
   ```batch
   "D:\迅投极速交易终端 睿智融科版\bin.x64\python.exe" auto_scheduler.py
   ```
   或尝试其他路径：
   ```batch
   "D:\迅投极速交易终端 睿智融科版\mpython\python.exe" auto_scheduler.py
   ```

### 问题 2：BAT 文件执行失败

**症状**：日志显示 `[错误] 文件不存在`

**解决**：
1. 确认三个 BAT 文件在项目根目录
2. 检查文件名是否完全匹配（包括中文字符）

### 问题 3：周末仍然执行

**症状**：周六或周日也执行了启动序列

**解决**：
1. 检查系统日期和时间是否正确
2. 查看日志中的 `[跳过] 今天是周末` 消息

### 问题 4：执行时间不准确

**症状**：没有在 8:30 准时执行

**解决**：
1. 检查系统时间是否同步
2. 查看日志中的 `[等待] 下次执行时间` 消息
