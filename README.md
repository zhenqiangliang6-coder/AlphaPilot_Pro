## 🛡️ V8.95 动态止损详解（保守版）

### 设计理念

针对**风险厌恶型投资者**和**震荡市场环境**，采用更敏感的止损阈值：
- 更早预警（-0.8% vs -1.5%）
- 更快减仓（-1.7% vs -2.5%）
- 更严清仓（-3.5% vs -4.0%）

**核心价值**：宁可小幅止损，绝不深套

### 工作流程

```
📊 持仓监控流程（保守版）：

初始状态
   ↓
亏损达到 -0.8% → 【进入监控状态】记录最低盈亏
   ↓
亏损达到 -1.7% → 【一级止损】卖出50%仓位
   ↓              ↓
   ↓         若反弹超过成本价 → 【重置状态】重新开始监控
   ↓
亏损达到 -3.5% → 【二级止损】全部清仓
```

### 三大核心机制

#### 1️⃣ 监控阶段（-0.8%触发）
- **目的**：极早期预警，密切跟踪
- **动作**：记录股票进入风险观察名单
- **优势**：对微小波动保持高度警惕

#### 2️⃣ 一级止损（-1.7%减半）
- **目的**：快速控制风险
- **动作**：卖出50%仓位，锁定小部分损失
- **优势**：在亏损扩大前果断行动

#### 3️⃣ 二级止损（-3.5%清仓）
- **目的**：严格止损，防止恶化
- **动作**：全部清仓，彻底离场
- **优势**：将最大亏损控制在3.5%以内

### 🔄 反弹保护机制

**场景示例**：
```
某股票成本价 100元
→ 下跌至 98.3元（-1.7%），触发一级止损，卖出50%
→ 随后反弹至 100.5元（+0.5%）
→ 【自动重置】止损状态清零，剩余50%仓位继续享受上涨
```

**核心价值**：
- ✅ 避免"卖在最低点"的遗憾
- ✅ 给予优质股票二次机会
- ✅ 平衡风险控制与收益捕捉

### 时间窗口优化

| 时间段 | 状态 | 原因 |
|--------|------|------|
| 09:30 - 10:45 | ⏸️ 暂停 | 避开开盘剧烈波动，防止误触发 |
| 10:45 - 14:50 | ✅ 执行 | 正常交易时段，流动性充足 |
| 14:50 - 15:00 | ⏸️ 暂停 | 避开尾盘集合竞价，流动性差 |

### 适用场景

✅ **适合**：
- 风险厌恶型投资者
- 震荡市、弱势市场
- 高波动性股票（科创板、创业板）
- 短线交易者

❌ **不适合**：
- 长线价值投资者
- 强势牛市环境
- 低波动蓝筹股（可能频繁触发）

## 💡 常见问题

### Q: 为什么不能用系统的 Python？
A: 本项目依赖 `xtquant` 模块，这是 QMT 交易终端的专有接口，只在 QMT 自带的 Python 环境中可用。

### Q: 如何调整止损阈值？
A: 修改 `config/settings.py` 中的三个参数：
- `STOP_LOSS_MONITOR_THRESHOLD`：监控触发点（当前0.008即-0.8%）
- `STOP_LOSS_LEVEL1_THRESHOLD`：一级止损点（当前0.017即-1.7%）
- `STOP_LOSS_LEVEL2_THRESHOLD`：二级止损点（当前0.035即-3.5%）

### Q: 保守版和平衡版有什么区别？
A: 
| 参数 | 保守版（V8.95） | 平衡版（V8.9） |
|------|----------------|---------------|
| 监控触发 | -0.8% | -1.5% |
| 一级止损 | -1.7% | -2.5% |
| 二级止损 | -3.5% | -4.0% |
| 适用场景 | 震荡市/熊市 | 震荡市/牛市 |
| 风险偏好 | 保守型 | 平衡型 |

### Q: 反弹保护会不会导致反复触发止损？
A: 不会。反弹保护要求股价**超过成本价**（盈利>0%）才会重置，这是一个较高的门槛，避免了小幅反弹导致的频繁切换。

### Q: 为什么不在开盘就执行止损？
A: 开盘前15分钟（09:30-09:45）市场波动剧烈，容易出现假突破。延迟到10:45执行可以过滤掉大部分噪音，提高止损准确性。

### Q: 如何更新代码？
A: 
1. 停止正在运行的策略（Ctrl+C）
2. 修改代码
3. 重新运行 `启动AlphaPilot.bat`（会自动清理缓存）

### Q: 信号文件放在哪里？
A: 放入 `signals/` 目录，程序会自动处理并移动到 `signals/processed/`

### Q: 如何查看实时日志？
A: 打开 `logs/` 目录中最新的日志文件
