# 分级量比阈值策略 - V2.3

## 🎯 核心改进

### 问题背景

**原逻辑的缺陷** ❌：
```
Day 2-N (等待期): 量比≥18.0 → 提前触发买入 ⭐⭐⭐
Day N (目标日):   量比≥18.0 → 才用信号价买入 ⚠️ 门槛太高！

场景：
- 目标日当天必须买入（保底机制）
- 但信号量比只有 5.0（远低于 18.0）
- 结果：不用信号价，改用实时价（可能更高）
- 错失低价机会！
```

---

## ✅ V2.3 解决方案

### 分级量比阈值

```json
{
    "603538": {
        "name": "美诺华",
        "type": "delayed",
        "delay_days": 2,
        "min_volume_ratio": 18.0,      // 等待期高标准
        "target_day_min_vr": 0.2       // 目标日低标准 ⭐
    }
}
```

**执行逻辑**：
```
Day 2-N (等待期): 
   ├─ 量比≥18.0 → 提前触发买入 ⭐⭐⭐
   └─ 严格筛选，确保强势

Day N (目标日):   
   ├─ 量比≥0.2 → 用信号价买入 ⭐⭐⭐
   └─ 低门槛，确保能用信号价（最优买点）
```

---

## 💡 设计理念

### 为什么目标日可以降低量比门槛？

1. **当天必须买入** ⭐⭐⭐
   - 目标日已到，无论如何都要执行
   - 不是"要不要买"的问题
   - 而是"用什么价格买"的问题

2. **找最优买点** ⭐⭐
   - 有信号就用信号价（通常更低）
   - 即使量比只有 0.2，也比实时价好
   - 信号价反映了主力的真实意图

3. **避免踏空** ⭐
   - 如果要求量比≥18.0
   - 很多目标日信号会被忽略
   - 错失低价机会

#### 目标日（check_and_execute）

```python

# 检查今日信号（使用低门槛）

if today_signal and today_signal.get('price', 0) > 0:

    # 使用信号价买入
    _execute_target_buy(code, stock_name, signal_price, "TARGET_DAY_SIGNAL")
```
