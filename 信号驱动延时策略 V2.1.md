# 信号驱动延时策略 - V2.1（目标日期增强版）

## 🎯 核心理念

### 传统延时策略（机械式）❌
```
Day 1: 收到信号 → 加入观察名单 → 等待 N 天
Day N: 时间到了 → 直接买入（不管有没有信号）

问题：
- 机械执行，不够灵活
- 可能错过当天的最佳买点
- 如果当天有更低价格，买不到
```

### V2.1 信号驱动增强版 ✅
```
Day 1: 收到信号 → 加入观察名单 → 开始等待 N 天

Day 2~N-1: 每天监听
   - 如果有新 BUY 信号 + 量比达标 → 立即买入 ⭐（提前触发）
   - 如果没有信号 → 继续等待

Day N (目标日期): 【关键改进】
   ① 先检查：今天有没有 BUY 信号？
      ✓ 有 → 用信号价格买入 ⭐（确保最低点，优先）
      ✗ 没有 → 用实时价格买入（保底机制）

结果：
- ✅ 确保慢涨股一定能买入（不会漏掉）
- ✅ 优先使用信号价（通常更低）
- ✅ 没有信号也有保底（市价买入）
```

## 🎯 三种买入时机

### 时机 1：提前触发（最优）⭐⭐⭐
```
条件：
- 在等待期内（Day 2 ~ N-1）
- 再次出现 BUY 信号
- 量比达到阈值

优势：
- ✅ 通常抓到最低点
- ✅ 确认主力持续介入
- ✅ 双重验证最可靠

日志关键词：
"等待第 N 天再次出现强势信号"
"提前买入成功"
```

### 时机 2：目标日期有信号（次优）⭐⭐
```
条件：
- 到达目标日期（Day N）
- 当天有 BUY 信号

优势：
- ✅ 使用信号价（通常较低）
- ✅ 确保当天能买入
- ✅ 不会错过行情

日志关键词：
"检测到今日新信号"
"优先使用信号价买入"
```

### 时机 3：目标日期无信号（保底）⭐
```
条件：
- 到达目标日期（Day N）
- 全天无新信号

特点：
- ✅ 保底机制，确保买入
- ⚠️ 使用实时价（可能较高）
- ✅ 不会踏空

日志关键词：
"未检测到今日新信号"
"使用实时价格买入"
```

def _get_today_signal_price(self, code):
    """获取今日最新信号价格"""
    
    # 1. 读取今日所有信号文件
    files = get_today_signal_files()
    
    # 2. 遍历查找该股票的 BUY 信号
    for file in files:
        signals = parse_signals(file)
        
        for sig in signals:
            if sig['code'] == code and sig['action'] == 'BUY':
                return sig['price']  # 找到返回
    
    return None  # 没找到

## 📞 常见问题

### Q1: 为什么目标日期还要等信号？

**A**: 因为慢涨股的特征就是**反复震荡上涨**，经常会出现：
- Day 1: 放量突破（信号触发）
- Day 2-3: 缩量洗盘（无信号或弱信号）
- Day 4: 再次放量（新信号，通常是较好买点）

**如果不等信号直接买**：
- ❌ 可能买到洗盘高点
- ❌ 错过再次放量的低吸机会
- ❌ 成本偏高

**如果等信号**：
- ✅ 用信号价买入（通常较低）
- ✅ 确认主力再次介入
- ✅ 成本更有优势

---

### Q2: 如果目标日期一整天都没信号怎么办？

**A**: 有保底机制！
```
Day N (目标日期):
  上午：等待信号 → 无
  下午：继续等待 → 无
  收盘前：仍然无信号
  
  → 启动保底机制
  → 使用实时价格买入
  → 确保不会踏空
```

**保底的好处**：
- ✅ 不会错过行情（即使没信号也买入）
- ✅ 完成策略闭环（有始有终）
- ✅ 避免踏空风险
