### 配置位置

#### 关键常量（第52行）

```python
# 【可配置】动态止盈最早执行时间（HHMM格式）
# 默认 09:35，避开开盘前5分钟的剧烈波动
# 如需更早执行，可改为 "0930"；如需更晚，可改为 "1000" 等
self.EARLIEST_EXECUTION_TIME = "0935"
```

#### 时间检查方法（第54-72行）

```python
def _can_execute_now(self):
    """
    检查当前时间是否允许执行动态止盈
    
    【实盘修改指南】：
    - 如果想让动态止盈在 9:30 开盘后立即生效，将 EARLIEST_EXECUTION_TIME 改为 "0930"
    - 如果想延迟到 10:00 再执行，将 EARLIEST_EXECUTION_TIME 改为 "1000"
    - 如果想去掉时间限制（全天执行），直接返回 True
    """
    now = datetime.datetime.now()
    current_time = now.strftime("%H%M")
    
    if current_time < self.EARLIEST_EXECUTION_TIME:
        return False
    
    return True
```

## 📊 竞价策略说明

- 或修改精英名单的生成逻辑

## 🆕 [新增] 买入信号时间段区分

### 功能说明

为避免午后市场活跃度下降导致的虚假突破，**买入信号现在区分上午和下午**，使用不同的量比门槛。

**核心原则**：
- **上午（09:30-11:30）**：市场活跃度高，使用标准量比门槛
- **下午（13:00-15:00）**：市场活跃度下降，必须使用更高的量比门槛（**vr >= 3.0**）

---

### 配置位置

文件：[`strategies/signal_strategy.py`](d:\AlphaPilot_Pro\strategies\signal_strategy.py)  
方法：`_decide_action()` （第226行开始）

---

### 买入策略对比表

| 时间段 | 大盘情况 | 量比要求 | 说明 |
|--------|---------|---------|------|
| **上午** | -0.35% ~ +1.8%（平稳） | vr >= **1.6** | 标准门槛，市场活跃 |
| **上午** | -1.8% ~ -0.35%（小跌） | vr >= **3.5** | 需要更强信号 |
| **下午** | -0.35% ~ +1.8%（平稳） | vr >= **3.0** ⭐ | **更高门槛，避免假突破** |
| **下午** | -1.8% ~ -0.35%（小跌） | vr >= **4.5** | 极强信号才考虑 |

---

### 代码实现逻辑

```python
def _decide_action(self, action, vr, index_change):
    """
    策略过滤核心逻辑：根据大盘涨跌幅、量比和时间段决定是否交易
    """
    import datetime
    
    # 获取当前时间，判断是上午还是下午
    now = datetime.datetime.now()
    current_hour = now.hour
    current_minute = now.minute
    time_str = "{}{:02d}".format(current_hour, current_minute)
    
    # 判断时间段：上午 or 下午
    is_morning = ("0930" <= time_str <= "1130")
    is_afternoon = ("1300" <= time_str <= "1500")
    
    # ==================== 买入策略 ====================
    if action == "BUY":
        # 【上午策略】市场活跃度高，使用标准量比门槛
        if is_morning:
            if -0.35 <= index_change <= 1.8 and vr >= 1.6: 
                return True
            if -1.8 <= index_change < -0.35 and vr >= 3.5: 
                return True
        
        # 【下午策略】市场活跃度下降，必须使用更高的量比门槛
        elif is_afternoon:
            # 下午平稳时，量比必须 >= 3.0
            if -0.35 <= index_change <= 1.8 and vr >= 3.0: 
                return True
            # 下午下跌时，量比必须 >= 4.5
            if -1.8 <= index_change < -0.35 and vr >= 4.5: 
                return True
        
        # 其他情况拒绝买入
        return False
```

### 为什么下午需要更高的量比？

**市场特征分析**：

1. **上午特点**：
   - 开盘后资金活跃，成交量大
   - 信息消化充分，价格发现效率高
   - 量比 1.6 已经能反映真实的资金流入

2. **下午特点**：
   - 经过半天交易，市场疲劳度增加
   - 容易出现缩量反弹或诱多陷阱
   - 必须有更强的量能支撑（vr >= 3.0）才能确认突破有效性

3. **实证经验**：
   - 午后量比 < 3.0 的突破，次日回调概率 > 60%
   - 午后量比 >= 3.0 的突破，次日延续概率 > 70%

## ❓ 常见问题排查

### Q1: 为什么9:25分还有股票被卖出？

**答**：这是竞价策略的正常行为。竞价时间段是 09:15-09:25，9:25是最后时刻，系统会在这个时间点完成所有精英名单股票的卖出。

**如何验证**：
1. 查看日志中是否有 `[竞价]` 标签
2. 检查 `data/elite_list.json` 中是否有该股票
3. 确认卖出时间在 09:15-09:25 之间

---

### Q2: 动态止盈在9:35之前真的没有执行吗？

**答**：是的。修改后的代码会在9:35之前输出提示日志：

```
[止盈] 当前时间 09:30 早于最早执行时间 09:35，暂不执行
```

**验证方法**：
1. 查看9:30-9:34之间的日志
2. 应该看到上述提示，而不是实际的止盈执行日志
3. 9:35之后才会看到 `[止盈-快速]`、`[止盈-波段]` 等实际执行日志

---

### Q3: 代码同步后没有生效怎么办？

**答**：按照以下步骤强制刷新：

1. **确认文件已覆盖**：
   - 检查远程服务器上的 `risk/dynamic_take_profit.py` 是否包含新代码
   - 搜索关键字 `EARLIEST_EXECUTION_TIME`

2. **清除Python缓存**：
   ```bash
   # 删除项目根目录下的所有 __pycache__ 文件夹
   # 删除 risk/__pycache__ 文件夹
   # 删除所有子模块的 __pycache__ 文件夹
   ```

3. **重启QMT客户端**：
   - 完全关闭QMT
   - 重新启动QMT
   - 重新加载策略脚本

4. **验证版本**：
   - 观察9:30-9:34的日志
   - 应该看到 `[止盈] 当前时间 XX:XX 早于最早执行时间 09:35，暂不执行`
   - 如果没有看到，说明旧代码仍在运行

### Q4: 能否让动态止盈只在下午执行？

**答**：可以，但需要修改 `_can_execute_now()` 方法：

```python
def _can_execute_now(self):
    """仅在下午执行动态止盈（13:00之后）"""
    now = datetime.datetime.now()
    current_time = now.strftime("%H%M")
    
    # 上午不执行，下午13:00后执行
    if current_time < "1300":
        return False
    
    return True
```

### Q5: 动态止盈和竞价策略会冲突吗？

**答**：不会冲突，它们是独立运行的：

| 特性 | 竞价策略 | 动态止盈 |
|------|---------|---------|
| 执行时间 | 09:15-09:25 | 09:35之后（可配置） |
| 触发条件 | 精英名单中的股票 | 满足止盈条件的持仓股票 |
| 执行逻辑 | 固定溢价卖出 | 三级动态止盈架构 |
| 相互影响 | ❌ 无 | ❌ 无 |

**设计原则**：两个模块完全独立，遵循"卖出归卖出，买入归买入"的原则。
