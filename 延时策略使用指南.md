# 慢涨股延时策略使用指南

## 一、策略背景

### 核心痛点
现有系统对所有信号"即时执行",导致部分"慢涨型"个股（如**美诺华**）在信号出现当天不涨，需等待 2-3 天洗盘后才涨停。现有逻辑无法捕捉此类机会，且容易混入无效卖出信号。

### 解决方案
引入"延时买入系统",通过以下机制解决问题:
1. **信号清洗**: 过滤 SELL 信号和低量比信号
2. **记忆持久化**: 记录符合条件的 BUY 信号到观察名单
3. **延时执行**: 等待 N 天后自动买入
4. **人机交互**: 支持手动配置个股参数

## 三、配置文件详解

### 1. stock_personalities.json (个股性格配置)

**用途**: 定义每只股票的"交易性格"和参数  
**维护方式**: 手动编辑，支持热更新 (修改后无需重启程序)

**完整示例**:
```json
{
    "603538": {
        "name": "美诺华",
        "type": "delayed",
        "delay_days": 3,
        "min_volume_ratio": 18.0
    },
    "300123": {
        "name": "田中精机",
        "type": "delayed",
        "delay_days": 4,
        "min_volume_ratio": 20.0
    },
    "002456": {
        "name": "欧菲光",
        "type": "immediate",
        "min_volume_ratio": 2.0
    },
    "default": {
        "type": "immediate",
        "min_volume_ratio": 1.5
    }
}
```

**字段说明**:
- `code` (键): 股票代码
- `name`: 股票名称 (便于识别)
- `type`: 
  - `"delayed"`: 慢涨股，需要等待 N 天
  - `"immediate"`: 即时股，按原逻辑处理
- `delay_days`: 延时天数 (仅 type=delayed 时有效)
- `min_volume_ratio`: 最小量比阈值 (低于此值的信号会被过滤)
- `default`: 未配置股票的默认参数

**配置技巧**:
1. **识别慢涨股**: 历史走势显示信号后 2-3 天才涨停的股票
2. **设置 delay_days**: 根据历史洗盘天数设定 (通常 2-4 天)
3. **设置 min_volume_ratio**: 慢涨股通常要求更高量比 (建议≥18)
4. **动态调整**: 可根据实际表现随时修改配置

---

### 2. delayed_watchlist.json (延时观察名单)

**用途**: 记录已触发信号、正在等待买入的股票  
**维护方式**: 系统自动读写，禁止手动修改

**数据结构**:
```json
{
    "last_update": "2026-04-04 15:30:25",
    "watchlist": {
        "603538": {
            "name": "美诺华",
            "action": "BUY",
            "signal_date": "2026-03-31",
            "target_date": "2026-04-03",
            "trigger_price": 40.59,
            "trigger_volume_ratio": 21.52,
            "status": "waiting",
            "delay_days": 3
        }
    }
}
```

**关键字段**:
- `signal_date`: 信号触发日期
- `target_date`: 目标买入日期 (= signal_date + delay_days)
- `trigger_price`: 触发时的价格
- `trigger_volume_ratio`: 触发时的量比
- `status`: 
  - `"waiting"`: 等待中
  - (执行成功后会自动从名单删除)

**重要规则**:
⚠️ **只记录 BUY 信号，严禁记录 SELL 信号**

## 四、策略执行流程

### 完整流程图

```
信号文件到达
    ↓
读取 JSON 信号
    ↓
判断：是 SELL 信号？
    ├─ YES → 直接忽略 (绝不写入观察名单) ✓
    └─ NO → 继续
    ↓
判断：是 BUY 信号？
    ├─ NO → 丢弃
    └─ YES → 继续
    ↓
读取 stock_personalities.json
    ↓
获取该股配置：
    - type (delayed/immediate)
    - min_volume_ratio
    - delay_days
    ↓
判断：当前量比 >= min_volume_ratio？
    ├─ NO → 丢弃 (量比不足) ✓
    └─ YES → 继续
    ↓
判断：type == "delayed"？
    ├─ NO → 交给 signal_strategy 即时处理
    └─ YES → 继续
    ↓
计算 target_date = signal_date + delay_days
    ↓
写入 delayed_watchlist.json (持久化记忆) ✓
    ↓
[系统持续运行]
    ↓
每日轮询观察名单
    ↓
判断：当前日期 >= target_date？
    ├─ NO → 继续等待
    └─ YES → 执行买入 ✓
    ↓
买入成功 → 从名单删除 ✓
```

## 七、配置技巧与建议

### 1. 如何识别慢涨股？

**特征**:
- 信号当天涨幅<3%，甚至下跌
- 后续 2-3 天内出现涨停
- 量比通常较高 (≥18)
- 洗盘特征明显 (小幅震荡整理)

**方法**:
1. 回测历史信号，统计表现
2. 标记那些"信号后 N 天才涨停"的股票
3. 加入 stock_personalities.json，配置 type="delayed"

### 2. delay_days 设置建议

| 股票类型 | 建议天数 | 说明 |
|---------|---------|------|
| 超强势股 | 0-1 天 | 当天或次日就涨 |
| 中等强势 | 1-2 天 | 洗盘 1 天后拉升 |
| 慢涨股 | 2-4 天 | 典型的美诺华模式 |
| 超慢涨股 | 5-7 天 | 较少见，需谨慎 |

### 3. min_volume_ratio 设置建议

| 类型 | 量比阈值 | 适用场景 |
|-----|---------|---------|
| 普通即时股 | 1.5-2.0 | 常规信号 |
| 强势股 | 3.0-5.0 | 强势突破 |
| 慢涨股 | 15.0-25.0 | 美诺华模式 |

### 4. 动态调整策略

**每周复盘**:
1. 检查 delayed_watchlist.json 中的成功案例
2. 分析失败案例 (买入后下跌的)
3. 调整 delay_days 和 min_volume_ratio
4. 将表现不佳的股票改为 type="immediate"

## 八、常见问题

### Q1: 如何确认延时策略是否生效？

**A**: 查看日志文件，搜索关键词:
- "已加入观察名单" → 信号被接收
- "到达目标日期" → 执行买入
- "成功买入" → 下单成功

### Q2: 配置文件修改后需要重启吗？

**A**: **不需要！** 系统会自动检测文件变更并重新加载。

### Q3: 如果程序重启，观察名单会丢失吗？

**A**: **不会！** 观察名单持久化保存在 JSON 文件中，重启后自动恢复。

### Q4: 周末和节假日如何处理？

**A**: 系统会自动跳过非交易日，只在交易日计数。

### Q5: 可以同时配置多只慢涨股吗？

**A**: **可以！** 理论上没有数量限制。

### Q6: 如果买入失败怎么办？

**A**: 系统会记录失败原因，该股票保留在观察名单中，下次循环继续尝试。

## 九、风险控制

### 1. 仓位控制
延时策略使用与 signal_strategy 相同的仓位管理:
- 单只股票最大仓位 = 总资产 × 50% ÷ 20 = 2.5%
- 自动计算可买数量

### 2. 现金检查
执行前会检查可用资金，不足时自动调整数量或跳过。

### 3. 防重复买入
买入成功后立即从名单删除，防止重复执行。

### 4. 价格保护
使用溢价 1% 确保成交，但不会超过涨跌停限制。

## 十二、总结

### 核心价值
✅ **捕捉慢涨机会**: 不再错过美诺华式的慢涨股  
✅ **过滤无效信号**: SELL 信号和低量比信号被严格过滤  
✅ **智能延时**: 自动等待洗盘完成后再买入  
✅ **人机协作**: 人工配置参数，系统自动执行  

### 适用场景
📌 信号当天不涨，后续 N 天才涨停的股票  
📌 需要洗盘整理的慢涨型个股  
📌 高量比但短期不启动的股票  

### 不适用场景
❌ 当天就涨停的强势股 (应使用即时策略)  
❌ 低量比的弱势股  
❌ SELL 信号
