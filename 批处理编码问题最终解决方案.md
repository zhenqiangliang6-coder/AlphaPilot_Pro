# 批处理脚本编码问题最终解决方案（2026-04-15）

## 📋 问题根源

**症状**: 
```
'杩呮姇鏋侀€熶氦鏄撶粓绔?鐫挎櫤铻嶇鐗?echo' 不是内部或外部命令
'n.exe"' 不是内部或外部命令
```

**根本原因**: 
Windows批处理文件中的**中文字符路径**在不同编码环境下会被错误解析，即使使用 `chcp 65001` 设置UTF-8编码，某些系统配置仍会导致乱码。

---

## ✅ 最终解决方案

### 核心技术：使用变量存储中文路径

**关键改进**：将中文路径存储在环境变量中，而不是直接在命令中使用。

#### ❌ 之前的写法（会出错）
```batch
if exist "D:\迅投极速交易终端 睿智融科版\bin.x64\python.exe" (
    set "QMT_PYTHON=D:\迅投极速交易终端 睿智融科版\bin.x64\python.exe"
)
```

#### ✅ 现在的写法（正确）
```batch
set "QMT_BASE=D:\迅投极速交易终端 睿智融科版"

if exist "%QMT_BASE%\bin.x64\python.exe" (
    set "QMT_PYTHON=%QMT_BASE%\bin.x64\python.exe"
)
```

### 为什么这样可以解决问题？

1. **变量定义时**：`set` 命令对编码更宽容
2. **变量使用时**：`%QMT_BASE%` 展开为ASCII字符，不会被编码问题影响
3. **引号保护**：所有路径都用双引号包裹，防止空格截断

## 🔍 技术细节

### Windows批处理编码机制

| 编码方式 | 代码页 | 特点 |
|---------|--------|------|
| **GBK** | 936 | Windows中文系统默认 |
| **UTF-8** | 65001 | 国际化标准，但兼容性有问题 |
| **ASCII** | - | 纯英文，完全兼容 |

### 我们的策略

1. **设置UTF-8编码**：`chcp 65001 >nul`
2. **使用英文提示**：避免中文输出的编码问题
3. **变量存储中文路径**：利用变量的编码容错性
4. **引号保护路径**：防止空格和特殊字符
